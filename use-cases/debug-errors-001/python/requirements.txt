## Error Description

The error `IndexError: list index out of range` occurs when the program tries to access a list element using an index that does not exist. In Python, list indexing starts at 0 and ends at `len(list) - 1`. Accessing anything beyond this range causes a crash.

In this case, the program tries to access one extra index beyond the list size.

---

## Root Cause

The issue is caused by an incorrect loop boundary:

```python id="loop_error"
for i in range(len(items) + 1):
````

The `+ 1` makes the loop run one extra time, causing `i` to exceed valid list indexes.

Example:

* List size = 3
* Valid indexes = 0, 1, 2
* Loop runs = 0, 1, 2, 3
* Index 3 does not exist → error occurs

---

## Solution

### Fix 1 (Correct loop range)

```python id="fix1"
for i in range(len(items)):
    print(f"Item {i+1}: {items[i]['name']} - Quantity: {items[i]['quantity']}")
```

### Fix 2 (Better approach using enumerate)

```python id="fix2"
for i, item in enumerate(items):
    print(f"Item {i+1}: {item['name']} - Quantity: {item['quantity']}")
```

---

## Learning Points

* Python lists are zero-indexed
* `range(len(list))` prevents out-of-bounds errors
* Off-by-one errors are very common in loops
* Using `enumerate()` is safer and more readable than manual indexing
* Always validate loop boundaries when working with lists
